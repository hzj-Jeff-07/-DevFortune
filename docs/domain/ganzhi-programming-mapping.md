# 五行与编程概念映射手册

> 天干地支五行理论在程序员日常工作中的完整映射参考。本文档是 [ADR-004](../adr/ADR-004-data-driven-mapping.md) 的配套实施手册。

## 映射设计原则

1. **意象联想**：映射基于五行的核心意象与编程活动的内在相似性，而非生搬硬套。
2. **可配置**：所有映射以 JSON 存储于 `packages/core/data/mappings/`，支持社区贡献和自定义（见 ADR-004）。
3. **文化自洽**：每个映射都能追溯到五行理论的文化根源，不是随意指定。

## 详细映射说明

### 🌳 木（Wood）— 生发与创造

**文化根源**：木主春季，万物萌发，生机勃勃。甲木为参天大树，乙木为花草藤蔓。木的核心特质是「生」与「长」。

**编程映射**：

| 维度 | 映射内容 | 文化依据 |
|------|----------|----------|
| **活动** | 新功能开发、架构搭建、原型设计、创意实验 | 木主生发 — 从无到有的创造过程 |
| **语言** | JavaScript、Python、Swift | 灵活多变、快速成长，如春天万物竞生 |
| **工具** | 脚手架工具（Vite/CRA）、原型工具（Figma）、IDE | 助力从零到一的创建工具 |
| **幸运活动** | 创建新项目、搭建脚手架、写技术方案、学习新框架 | 顺应木之生发方向 |
| **忌讳活动** | 大规模删除代码、砍掉功能模块、做减法优化 | 逆木之生意，金克木 |

**旺衰影响**：
- **木旺（春季/寅卯辰月）**：创造力旺盛，适合启动新项目
- **木弱（秋季/申酉戌月）**：金旺克木，新想法容易夭折，宜蛰伏积累

---

### 🔥 火（Fire）— 光明与传播

**文化根源**：火主夏季，光明炎上，温暖万物。丙火为太阳，丁火为灯烛。火的核心特质是「明」与「烈」。

**编程映射**：

| 维度 | 映射内容 | 文化依据 |
|------|----------|----------|
| **活动** | 性能优化、Code Review、团队协作、技术分享 | 火主光明 — 照亮问题、传播知识 |
| **语言** | Rust、C++、Assembly | 追求极致性能，如烈火淬炼 |
| **工具** | 性能分析器（Profiler）、火焰图、压测工具 | 「火」焰图之名恰如其分 |
| **幸运活动** | 优化热路径、做技术分享、组织 Code Review | 顺应火之照耀传播特性 |
| **忌讳活动** | 仓促上线、跳过测试直接部署、独自闷头开发 | 火失控则为灾，需有序释放 |

**旺衰影响**：
- **火旺（夏季/巳午未月）**：精力充沛，适合攻克性能瓶颈
- **火弱（冬季/亥子丑月）**：水旺克火，容易忽视细节，Review 时需格外仔细

---

### 🌍 土（Earth）— 承载与稳定

**文化根源**：土主长夏（季夏），居中央，承载万物。戊土为城墙大地，己土为田园沃土。土的核心特质是「稳」与「载」。

**编程映射**：

| 维度 | 映射内容 | 文化依据 |
|------|----------|----------|
| **活动** | 基础设施搭建、数据库维护、CI/CD 建设、运维 | 土主承载 — 一切上层建筑的根基 |
| **语言** | Go、Java、SQL | 稳重可靠，适合承载型系统 |
| **工具** | Docker、Kubernetes、Terraform、数据库管理工具 | 基础设施即代码，如大地承载万物 |
| **幸运活动** | 稳固架构、完善 CI/CD、数据备份、写运维文档 | 顺应土之厚德载物 |
| **忌讳活动** | 激进迁移、未测试的 schema 变更、跳过灰度发布 | 土忌动荡，动则根基不稳 |

**旺衰影响**：
- **土旺（长夏/辰戌丑未月）**：基础工作事半功倍，适合做稳定性治理
- **土弱（春季/寅卯辰月）**：木旺克土，基础设施可能出问题，注意监控告警

---

### ⚔️ 金（Metal）— 收敛与精炼

**文化根源**：金主秋季，万物收敛，去芜存菁。庚金为刀剑斧钺，辛金为珠玉首饰。金的核心特质是「敛」与「锐」。

**编程映射**：

| 维度 | 映射内容 | 文化依据 |
|------|----------|----------|
| **活动** | 代码重构、技术选型评审、依赖清理、安全审计 | 金主肃杀 — 去粗取精，精简提炼 |
| **语言** | Rust、TypeScript、Haskell | 类型严格、安全至上，如金之精纯 |
| **工具** | SonarQube、ESLint、静态分析器、依赖审计工具 | 检测与裁剪的利器 |
| **幸运活动** | 删除冗余代码、精简依赖树、重构遗留系统 | 顺应金之收敛肃杀 |
| **忌讳活动** | 引入大量新依赖、盲目追新框架、增加系统复杂度 | 金忌膨胀，金少则不成器 |

**旺衰影响**：
- **金旺（秋季/申酉戌月）**：重构效率最高，适合偿还技术债务
- **金弱（夏季/巳午未月）**：火旺克金，重构容易引入新 Bug，宜小步迭代

---

### 💧 水（Water）— 智慧与流动

**文化根源**：水主冬季，万物闭藏，蓄势待发。壬水为大海江河，癸水为雨露泉水。水的核心特质是「智」与「润」。

**编程映射**：

| 维度 | 映射内容 | 文化依据 |
|------|----------|----------|
| **活动** | 学习新技术、阅读文档、写技术博客、探索开源 | 水主智慧 — 知识如水般流动渗透 |
| **语言** | Elixir、Clojure、Lisp | 流动性强、函数式思维，如水般灵活 |
| **工具** | 文档工具（Notion/Obsidian）、学习平台、开源社区 | 知识的载体与流通渠道 |
| **幸运活动** | 探索新技术、阅读源码、参与开源社区、写文档 | 顺应水之智慧流动 |
| **忌讳活动** | 闭门造车、拒绝新知识、独断技术选型 | 水忌堵塞，壅则溢 |

**旺衰影响**：
- **水旺（冬季/亥子丑月）**：学习能力最强，适合深度研究新技术
- **水弱（长夏/辰戌丑未月）**：土旺克水，容易浅尝辄止，宜专注一个方向深入

## 五行生克与编程场景

### 相生关系（协同）

| 生克 | 编程解读 | 实践建议 |
|------|----------|----------|
| 木生火 | 新功能开发（木）完成后自然进入性能优化（火） | 开发完 feature 后紧接着做性能 Review |
| 火生土 | 性能优化（火）沉淀为稳定的基础设施（土） | 优化成果固化到 CI/CD 和监控体系中 |
| 土生金 | 稳定的基础（土）上才能安全地做重构（金） | 有完善的测试和回归环境后再启动重构 |
| 金生水 | 代码精简（金）后思路清晰，有余力学习新知（水） | 技术债务清理完后适合做技术调研 |
| 水生木 | 新知识（水）积累充分后催生新创意（木） | 学习充分后启动新项目事半功倍 |

### 相克关系（冲突）

| 生克 | 编程解读 | 风险提示 |
|------|----------|----------|
| 木克土 | 大量新功能（木）动摇基础设施稳定性（土） | 快速迭代期注意基础设施承载力 |
| 土克水 | 繁重运维（土）挤占学习时间（水） | 避免陷入运维泥潭，保留学习空间 |
| 水克火 | 过度研究（水）导致行动迟缓、性能优化拖延（火） | 研究要设定时间盒，避免分析瘫痪 |
| 火克金 | 匆忙优化（火）破坏已有的精简架构（金） | 性能优化前先评估对代码结构的影响 |
| 金克木 | 过度重构（金）扼杀创新空间（木） | 重构不是目的，不要为了完美而阻碍新功能 |

## 如何贡献自定义映射

1. 复制 `packages/core/data/mappings/default.json` 为新文件
2. 修改 `id`、`name`、`description` 标识你的映射方案
3. 按你的理解调整 `themes`、`lucky`、`unlucky` 内容
4. 确保 JSON 通过 Schema 验证：`npx ajv validate -s schemas/mapping.schema.json -d your-mapping.json`
5. 提交 PR，附上你对映射方案的文化解读
